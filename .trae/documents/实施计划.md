# C语言自动批阅系统 - 实施计划

## 一、项目概述

基于大语言模型的C语言课后作业自动批阅系统，采用"静态分析 + LLM评阅 + RAG增强"三层联动架构。

**技术栈**：
- 前端：Vue 3 + TypeScript + Element Plus + Monaco Editor + ECharts + Pinia
- 后端：Spring Boot 3.x (Java 17) + MyBatis-Plus + JWT + Redis
- 批阅引擎：Python + Celery + ChromaDB + Sentence-Transformers
- 数据库：MySQL 8.0 + Redis 7.0 + Chroma向量库
- 基础设施：Docker + Nginx

## 三、实施顺序建议

### 建议1：后端先行
1. 数据库初始化 → 后端基础架构 → 核心业务API → 前端页面 → 批阅引擎

### 建议2：前后端并行
- 后端：基础模块先跑通，API接口稳定后再对接前端
- 前端：页面和组件并行开发，通过Mock数据调试

### 建议3：批阅引擎独立开发
- 引擎可独立测试，不依赖后端完成
- 通过Celery任务队列与后端解耦

## 六、接下来的具体步骤

### 步骤1：数据库初始化
```bash
# 创建数据库和表
docker exec autograding-mysql mysql -uroot -p123456 < docker/mysql/init.sql
# 导入种子数据
docker exec autograding-mysql mysql -uroot -p123456 < docker/mysql/seed.sql
```

### 步骤2：修改后端配置
- 更新`application.yml`中的数据库密码
- 配置JWT密钥和过期时间

### 步骤3：启动后端服务
```bash
java -jar backend/target/backend-1.0.0-SNAPSHOT.jar
```

### 步骤4：启动前端开发服务器
```bash
cd frontend && npm run dev
```

### 步骤5：启动Celery Worker
```bash
cd grading-engine && celery -A celery_app worker --loglevel=info
```
